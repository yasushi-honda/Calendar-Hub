# Calendar Hub - Product Requirements Document

## 1. プロダクトビジョン

複数のGoogleカレンダーアカウントとTimeTreeを一元管理し、AIが最適なスケジュールを提案する統合スケジュール管理アプリ。

**ミッション**: ユーザーの仕事と生活のバランスを尊重した、無理のないスケジュール管理を実現する。

## 2. ユーザーペルソナ（初期）

- **対象**: 自分（Single User）
- **課題**:
  - 複数のGoogleアカウント（仕事用・個人用）の予定を横断的に把握できない
  - TimeTree（家族共有）との予定重複チェックが手動
  - 空き時間の把握と有効活用ができていない
  - スケジュール調整に時間がかかる

## 3. 機能要件

### P0 - Must Have（MVP）

| ID    | 機能                     | 説明                                                                          |
| ----- | ------------------------ | ----------------------------------------------------------------------------- |
| F-001 | 複数Googleアカウント連携 | OAuth 2.0で複数のGoogleアカウントを連携し、各アカウントのカレンダー一覧を取得 |
| F-002 | Google Calendar CRUD     | 連携した各アカウントのカレンダーイベントの作成・取得・更新・削除              |
| F-003 | TimeTree連携             | TimeTree OAuth Appでカレンダー連携、イベントの取得・作成・更新・削除          |
| F-004 | 統合カレンダービュー     | 全カレンダーソースの予定を月/週/日ビューで統合表示。ソース別に色分け          |
| F-005 | 空き時間算出             | 全カレンダーの予定をマージし、空き時間スロットを自動算出・表示                |

### P1 - Should Have

| ID    | 機能                     | 説明                                                                     |
| ----- | ------------------------ | ------------------------------------------------------------------------ |

| F-102 | ユーザープロファイル設定 | 勤務時間、休日、集中時間帯、睡眠時間、食事時間等をAIが対話的にヒアリング |

| F-104 | 通知（Google Chat）      | スケジュールリマインダーやAI提案をGoogle Chat Webhookで通知              |
| F-105 | 通知（Email）            | Gmail APIでメール通知。タイミング・頻度はユーザー設定可能                |

### P2 - Nice to Have

| ----- | ------------------------ | --------------------------------------------------------------- |

## 4. 非機能要件

### パフォーマンス

- カレンダービュー表示: 2秒以内
- カレンダー同期: 最大30秒（初回）、差分同期5秒以内
- AI提案生成: 10秒以内

### セキュリティ

- OAuthトークンはFirestoreに暗号化して保存
- Workload Identity FederationでGCPサービス間認証（サービスアカウントキー不使用）
- HTTPS通信のみ
- CORS設定による不正アクセス防止

### 可用性

- Cloud Runのオートスケーリング（0→Nインスタンス）
- Firestoreの99.999% SLA活用
- コールドスタート最適化（5秒以内）

### コスト

- Vertex AI: Gemini 2.5 Flash（低コストモデル）を採用
- Cloud Run: 最小インスタンス0で未使用時のコスト発生なし
- Firestore: Sparkプラン（無料枠）から開始

| レイヤー         | 技術                                 |
| ---------------- | ------------------------------------ |
| Frontend         | Next.js 15 (App Router) + TypeScript |

| Event Processing | Cloud Functions (2nd gen)            |
| Database         | Firestore                            |
| AI               | Vertex AI Gemini 2.5 Flash (GA)      |
| Auth             | Firebase Auth + Google OAuth 2.0     |
| Notifications    | Google Chat API + Gmail API          |
| Calendar APIs    | Google Calendar API + TimeTree API   |
| CI/CD            | Cloud Build + GitHub Actions         |
| Auth Management  | Workload Identity Federation         |

## 6. 成功指標（初期）

| 指標               | 目標                                               |
| ------------------ | -------------------------------------------------- |
| カレンダー統合表示 | 全ソースの予定が正確に統合表示される               |
| CRUD操作           | 各カレンダーへのイベント作成・更新・削除が正常動作 |
| 空き時間算出       | 全カレンダーの予定を考慮した正確な空き時間表示     |
| AI提案精度         | ユーザー設定に基づく合理的な提案（主観評価）       |
| レスポンス時間     | カレンダービュー表示2秒以内                        |

## 7. リスクと緩和策

| リスク                        | 影響度 | 緩和策                                                      |
| ----------------------------- | ------ | ----------------------------------------------------------- |
| TimeTree API の仕様変更・廃止 | 高     | アダプターパターンで疎結合化。iCal/CalDAVフォールバック検討 |
| Google OAuth の同意画面審査   | 中     | 開発中はテストユーザー設定で回避。公開時に審査申請          |
| Gemini モデルのバージョン変更 | 低     | モデルID設定外部化。バージョン固定 + 定期的なアップデート   |
| OAuthトークン漏洩             | 高     | Firestore暗号化、Workload Identity、Secret Manager活用      |
